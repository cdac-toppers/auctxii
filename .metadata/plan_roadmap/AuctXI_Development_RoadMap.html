AuctXI

Professional Development Roadmap v1.0

This document outlines how AuctXI is built — not as a college project, but as if running a small software team. Every feature follows the same lifecycle: Jira Story → Figma Design → GitHub Branch → Development → Pull Request → Review → Testing → Merge → Deployment → Documentation.

Core Principle

Instead of thinking of this as "building a college project," treat it as running a small software team. If you consistently follow the professional workflow for every feature, you'll gain experience that closely resembles how many professional development teams work.

Recommendation
